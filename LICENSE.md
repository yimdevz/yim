## 🔧 Troubleshooting & Common Issues

### ❌ Menu Doesn't Open After Injection

**Symptoms:** Injection success toast appears but hotkey does nothing

**Solutions:**
- Confirm the game window is in focus (click on the game window first)
- Check `config.json` — ensure `menu_hotkey` isn't conflicting with a game keybind
- Try the secondary default hotkey `F4` as a fallback
- Re-run loader as Administrator

***

### ❌ Crash on Injection

**Symptoms:** Game closes immediately or freezes during loader execution

**Solutions:**
1. Verify your antivirus has whitelisted the YIM folder
2. Confirm you're running the correct build (Legacy vs Enhanced — don't mix them)
3. Delete `%APPDATA%\YIM\cache\` folder and retry
4. Ensure `.NET 6.0 Runtime` is installed from the Microsoft website

***

### ❌ Features Greyed Out in Online Mode

**Symptoms:** Several menu items appear disabled in online sessions

**Solutions:**
- This is **intentional** — certain features are restricted to story/private sessions for safety
- Switch to an invite-only or solo session to unlock full feature availability

***

### ❌ LUA Scripts Not Loading

**Symptoms:** Scripts placed in `/scripts` folder aren't executing

**Solutions:**
- Confirm `lua_autoload: true` in config.json
- Check that script filenames don't contain spaces or special characters
- Open the in-app LUA console (`Settings → Developer → Script Console`) for error output
- Verify script compatibility with LUA 5.4 — some older community scripts target 5.3
